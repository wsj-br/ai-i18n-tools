# عرض ميزة الترجمة

توجد هذه الصفحة لتوضيح كيفية تعامل `ai-i18n-tools` مع كل عناصر Markdown الشائعة. قم بتشغيل الأمر `sync` ضدها وقارن الناتج في كل مجلد من مجلدات اللغة المحلية لمعرفة بدقة ما يتم ترجمته وما يظل دون تغيير.

---

## النصوص العادية

الترجمة الدولية تتجاوز مجرد استبدال الكلمات. خط أنابيب الترجمة الجيد يحافظ على هيكل المستند، ويحافظ على المعرفات التقنية كما هي، ويرسل فقط النصوص القابلة للقراءة من قبل البشر إلى نموذج اللغة.

يقوم `ai-i18n-tools` بتقسيم كل مستند إلى **مقاطع** قبل إرسالها إلى نموذج اللغة الكبير (LLM). يتم ترجمة كل مقطع بشكل مستقل ثم إعادة تجميعه، وبالتالي فإن تغيير فقرة واحدة لا يجعل الترجمات المخزنة مؤقتًا لبقية الملف غير صالحة.

---

## التنسيق المضمن

يجب أن ينقل المترجم كل التنسيقات المضمنة دون تغيير العلامات:

- يجب أن يظل النص **العريض** عريضًا بعد الترجمة لأنه يشير إلى الأهمية.
- يستخدم النص _المائل_ للتأكيد أو العناوين؛ ويجب الحفاظ على المعنى.
- يُستخدم ~~النص المشطوب~~ للإشارة إلى المحتوى الذي تم إلغاؤه أو إزالته.
- لا يتم ترجمة `الكود المضمن` **أبدًا** — يجب أن تظل المعرفات وأسماء الدوال ومسارات الملفات كما هي.
- تحتفظ [الارتباط التشعبي](https://github.com/your-org/ai-i18n-tools) بعنوان URL الأصلي؛ ويتم ترجمة تسمية الرابط فقط.

## الجداول

تُعد الجداول مصدرًا شائعًا لأخطاء الترجمة. يتم ترجمة كل خلية بشكل منفصل؛ مع الحفاظ على فواصل الأعمدة وبناء الجدول الخاص بالمحاذاة.

| الميزة | الحالة | الملاحظات |
|---|---|---|
| ترجمة Markdown | ✅ مستقرة | يتم تخزين المقاطع مؤقتًا في SQLite |
| استخراج سلاسل واجهة المستخدم | ✅ مستقرة | يقرأ استدعاءات `t("…")` |
| سلاسل واجهة المستخدم الجمعية الأساسية | ✅ مستقرة | `t("…", { plurals: true, count })`؛ كتالوج + لواحق JSON مسطحة |
| ترجمة تسميات JSON | ✅ مستقرة | JSON شريط الجانبي/شريط التنقل في Docusaurus |
| ترجمة نصوص SVG | ✅ مستقرة | يحافظ على هيكل SVG |
| تطبيق القاموس | ✅ مستقرة | قاموس CSV لكل مشروع |
| التزامن الدفعي | ✅ قابل للتكوين | مفتاح `batchConcurrency` |

### أنواع المحاذاة

| محاذاة لليسار | متمركز | محاذاة لليمين |
|:---|:---:|---:|
| اللغة المصدر | `en-GB` | مطلوب |
| اللغات المستهدفة | حتى 20 | موصى به |
| التوازي | 4 | افتراضي |

## القوائم

### غير مرتبة

- يقوم ذاكرة الترجمة بتخزين تجزئة لكل مقطع مصدر.
- فقط المقاطع التي تغيرت تجزئتها منذ التشغيل الأخير يتم إرسالها إلى نموذج اللغة الكبير (LLM).
- هذا يجعل التشغيل التدريجي سريعًا جدًا — عادةً تتطلب التعديلات الصغيرة بضع استدعاءات فقط للواجهة البرمجية (API).

### مرتبة

1. أضف `ai-i18n-tools` كاعتماد تطوير.
2. أنشئ ملف `ai-i18n-tools.config.json` في جذر مشروعك.
3. شغّل `npx ai-i18n-tools sync` لإجراء الترجمة الكاملة الأولى.
4. قم بتأكيد ملفات اللغة المُنشأة مع ملفات المصدر.
5. في التشغيلات اللاحقة، يتم إعادة ترجمة المقاطع التي تغيرت فقط.

### متداخلة

- **خط أنابيب المستندات**
  - المصدر: أي ملف `.md` أو `.mdx`
  - المخرجات: هيكل شجري `i18n/` في Docusaurus أو نسخ مترجمة مسطحة
  - التخزين المؤقت: SQLite، مفتاحه هو مسار الملف + تجزئة المقطع
- **خط أنابيب سلاسل واجهة المستخدم**
  - المصدر: ملفات JS/TS مع استدعاءات `t("…")` (بما في ذلك الجمعيات الأساسية عبر `{ plurals: true, count }`)
  - المخرجات: ملفات JSON مسطحة لكل لغة (`de.json`، `fr.json`، …) مع مفاتيح ذات لواحق لفئات الجمع عند الحاجة
  - التخزين المؤقت: كتالوج `strings.json` الرئيسي نفسه

## سلاسل واجهة المستخدم الجمعية الأساسية (تطبيق Next.js المرافق)

تُظهر مستندات Markdown على هذا الموقع ترجمة **المستند**. من الأسهل ملاحظة السلوك **الجمعي الأساسي** لنصوص واجهة المستخدم في **مثال Next.js المجمّع** الموجود بجانب `docs-site/` ضمن `examples/nextjs-app/`.

تشتمل الصفحة الرئيسية لهذا التطبيق (`src/app/page.tsx`) على قسم **عرض الجمعيات** وتعيد عرض رسالة واحدة بعدد من العينات بحيث يمكنك مقارنة القواعد النحوية عبر اللغات (مثلًا العربية مقابل الإنجليزية). تستدعي كل سطر:

```typescript
t("This page has {{count}} sections", { plurals: true, count })
```

استخدم **`plurals: true`** بحيث يسجل **`extract`** مجموعة جمعية في `locales/strings.json` ويملأ **`translate-ui`** الملفات المسطحة لكل لغة ضمن `public/locales/`. أثناء التشغيل، يقوم i18next بحل المفتاح المُلحَق الصحيح للغة النشطة **`count`**؛ ويستخدم مثال Next.js أدوات مساعدة في **`src/lib/i18n.ts`**.

لعرض لقطات الشاشة وعناوين URL الخاصة باللغات وتخطيط الملفات، راجع **مثال الجمعيات الأساسية** في [دليل مثال Next.js](../../README.md).

## كتل التعليمات البرمجية

لا يتم ترجمة كتل التعليمات البرمجية **أبدًا**. يتم ترجمة النص المحيط، ولكن يتم تمرير كل حرف داخل الكتلة المحددة كما هو دون تغيير.

### shell

```bash
# Install the package
npm install --save-dev ai-i18n-tools

# Run a full sync
npx ai-i18n-tools sync

# Translate only documentation
npx ai-i18n-tools sync --no-ui --no-svg
```

### تهيئة JSON

```json
{
  "sourceLocale": "en-GB",
  "targetLocales": ["de", "es", "fr", "pt-BR"],
  "features": {
    "translateMarkdown": true,
    "translateJSON": true
  },
  "documentations": [
    {
      "contentPaths": ["docs-site/docs/"],
      "outputDir": "docs-site/i18n",
      "markdownOutput": { "style": "docusaurus", "docsRoot": "docs-site/docs" }
    }
  ]
}
```

### TypeScript

```typescript
import { createI18nConfig } from 'ai-i18n-tools/runtime';

const config = createI18nConfig({
  defaultLocale: 'en-GB',
  supportedLocales: ['de', 'es', 'fr', 'pt-BR'],
  fallback: 'en-GB',
});

export default config;
```

## التنبيهات (Docusaurus)

يتم ترجمة عناوين التنبيهات في Docusaurus؛ بينما تبقى أسوار `:::` والكلمات المفتاحية للنوع كما هي دون تغيير.

:::note
تم إعداد هذا المستند بشكل متعمد غنيًا بسمات Markdown. الغرض الأساسي منه هو أن يكون كموضوع اختبار للترجمة — قم بتشغيل `sync` وافحص المخرجات للتحقق من معالجة كل عنصر بشكل صحيح.
:::

:::tip
يمكنك تجاوز الصيغة المترجمة لأي مقطع عن طريق تعديل ملف الإخراج ثم تشغيل `sync` مرة أخرى. سيكتشف الأداة تعديلاتك وسيضيف الصيغة المصححة تلقائيًا إلى مسرد مصطلحات المشروع.
:::

:::warning
لا تقم بتأكيد دليل `.translation-cache/` في التحكم بالإصدار. إن ذاكرة التخزين المؤقت خاصة بالجهاز وتُعاد إنشاؤها في كل مرة يتم فيها استرجاع المشروع.
:::

:::danger
حذف دليل الذاكرة المؤقتة يؤدي إلى إعادة ترجمة جميع المقاطع من الصفر. قد يكون ذلك مكلفًا إذا كانت مستنداتك كبيرة. استخدم `sync --no-cache-write` لإجراء تشغيل تجريبي دون حفظ النتائج.
:::

## الصور وإعادة كتابة المسار حسب الموقع الجغرافي

يتم ترجمة نصوص وصف الصور إلى كل لغة محلية. وبالإضافة إلى ذلك، يمكن لأداة `ai-i18n-tools` أيضًا **إعادة كتابة مسارات الصور** في المخرجات المترجمة عبر `postProcessing.regexAdjustments` — بحيث يمكن لكل لغة محلية أن تشير إلى لقطة شاشة خاصة بها بدلاً من عرض النسخة الإنجليزية دائمًا.

الوثيقة المصدر (الإنجليزية) تشير إلى:

```markdown
![The example Next.js app running in English](/img/screenshots/ar/screenshot.png)
```

يحتوي إدخال التهيئة لهذا الموقع التوثيقي على:

```json
"regexAdjustments": [
  {
    "description": "Per-locale screenshot folders in docs-site static assets",
    "search": "screenshots/ar/",
    "replace": "screenshots/${translatedLocale}/"
  }
]
```

بعد الترجمة، تصبح المخرجات الألمانية كالتالي:

```markdown
![Die Beispiel-Next.js-App auf Deutsch](/img/screenshots/de/screenshot.png)
```

فيما يلي لقطة الشاشة الإنجليزية الفعلية — إذا كنت تقرأ هذا المحتوى بلغة محلية مترجمة، فيجب أن تعرض الصورة أدناه التطبيق بلغتك:

![The example Next.js app — UI strings and this page translated by ai-i18n-tools](/img/screenshots/ar/screenshot.png)

## الخطوط الأفقية وفواصل الأسطر

الخط الأفقي (`---`) هو عنصر هيكلي ولا يتم ترجمته.

يتم التعامل مع المحتوى فوقه وتحته كمقاطع منفصلة، مما يمنح نموذج اللغة الكبيرة (LLM) نوافذ سياق أنظف.
