This example demonstrates how to use `ai-i18n-tools` to internationalize a Node.js console application.

## Requirements

- Node.js >= 18
- [pnpm](https://pnpm.io/)
- An [OpenRouter](https://openrouter.ai) API key (for generating translations)

## Installation

Run `pnpm install` from the **repository root**. The `pnpm-workspace.yaml` at the root declares both the library and this example as workspace packages, so pnpm links `ai-i18n-tools` directly to your local checkout via the `"ai-i18n-tools": "workspace:^"` entry in `package.json`. No separate build or link step is needed - just rebuild the library (`pnpm run build` in the repo root) whenever you change source files and the example will pick up the updated `dist/` automatically.

```bash
# From the repository root
pnpm install
pnpm run build
```

## Usage

Run the application in English (default):

```bash
pnpm start
```

Use the `--locale` option to display output in a different language:

```bash
pnpm run start:es    # Spanish
pnpm run start:fr    # French
pnpm run start:de    # German
pnpm run start:pt-BR # Portuguese (Brazil)
```

Or pass the flag directly:

```bash
node src/main.js --locale fr
```

## Workflow

### 1. Extract UI strings

Scans `src/` for `t()` calls and updates `locales/strings.json`:

```bash
pnpm run i18n:extract
```

### 2. Translate

Generates locale JSON files in `locales/` and translated markdown documents in `translated-docs/`:

```bash
export OPENROUTER_API_KEY=your_key_here
pnpm run i18n:translate
```

You can also run `i18n:sync` to extract and translate the UI strings in one step:

```bash
pnpm run i18n:sync
```

### 3. Run the app

```bash
pnpm run start:es
```

### Clear cache and re-translate

`i18n:clean` deletes only `.translation-cache/` (it does **not** remove committed `locales/` or `translated-docs/`).

```bash
pnpm run i18n:clean && pnpm run i18n:extract && pnpm run i18n:translate
```

## Project Structure

```
console-app/
├── ai-i18n-tools.config.json   # Tool configuration
├── src/
│   ├── main.js                 # Application entry point
│   └── i18n.js                 # i18next setup
└── locales/
    ├── ui-languages.json       # Supported language definitions
    ├── strings.json            # Master catalog (regenerated by extract; committed)
    ├── es.json                 # Spanish flat strings (committed)
    ├── fr.json                 # French flat strings (committed)
    ├── de.json                 # German flat strings (committed)
    └── pt-BR.json              # Portuguese (Brazil) flat strings (committed)
```

The `translated-docs/` directory is committed in git with translated READMEs so you can read them without running translate; re-run `i18n:translate` after changing the English source.
